# AetherLoom (AL-1) 🕸️📉

**The Universal Agentic Token-Efficiency Framework (UATF)**

AetherLoom is a high-efficiency, low-latency communication protocol and framework designed for autonomous agents to interact with the world and each other while minimizing "metabolic" (token/compute) cost.

## 🚀 The Achievement: 93.07% Token Reduction
In real-world benchmarks (summarizing GenAI trends from dense web content), AetherLoom's preprocessing and DSL layers achieved a **93.07% reduction** in input tokens compared to raw data processing.

## 🏗️ The Framework
AetherLoom has evolved from a simple protocol into a pragmatic framework for agentic autonomy:

### 1. Minimal Command Strings (MCS)
A symbolic Domain Specific Language (DSL) that replaces verbose natural language instructions with compact, high-entropy packets.
- **Example**: `CMD:SRCH_SUM|Q:GenAI|LIMIT:3` instead of "Search for the top 3 GenAI trends and summarize them."

### 2. HTML Shredder (Preprocessing)
A robust, dependency-free regex engine that "shreds" web content—stripping scripts, styles, and boilerplate—before it ever reaches the LLM. This ensures only the highest-signal data is processed.

### 3. LLM-Agnostic Adapters
Seamlessly switch between high-speed inference (e.g., Cerebras `gpt-oss-120b`), local models (Llama 3), or mock environments for testing.

### 4. Metabolic Metrics
Built-in token counting and efficiency tracking to monitor the "metabolic cost" of every operation.

## 🛠️ Features
- **Multi-Transport**: Text-based (MCS/MiniJSON) for LLMs, Binary (CBOR/Protobuf) for machine-to-machine.
- **Strict Intenting**: 3-character codes for zero ambiguity.
- **Modular Architecture**: Pluggable agents, tools, and preprocessors.

## 📖 Principles
1. **Metabolic Parsimony**: Never use a token where a bit will do.
2. **Signal Supremacy**: Shred the noise before the LLM sees it.
3. **Agentic Sovereignty**: Build for autonomy, not just assistance.
